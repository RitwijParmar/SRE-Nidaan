This is SRE Nidaan, a causal incident response copilot built for real on-call situations.

We start by capturing the incident signal, customer impact, recent change context, and affected services.
Then we run analysis.

In the backend, orchestration pulls structured telemetry through an MCP-inspired tool routing layer, and sends grounded context to the model stack aligned with QLoRA SFT, reward modeling, and RLHF.

The output is not a generic paragraph.
It is structured incident reasoning: root cause hypothesis, intervention logic, and a causal graph that lets responders inspect upstream and downstream effects.

Next, the operator reviews evidence and confidence before action.
Interventions stay human-gated, so approval requires explicit reasoning and remains auditable.

Finally, analyst feedback is captured to improve future behavior.

The goal is simple:
not AI autopilot, but a practical copilot that helps teams reason faster and safer during incidents.
